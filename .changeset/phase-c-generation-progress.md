# Phase C: GenerationProgress + files in editor

Codesmith generation progress is now visible in real-time via a `GenerationProgress`
component using the `shared:generation-progress` surface. Files generated by the
`write_file` tool land directly in the editor pane (not as chat CodeBlocks).

A `SummaryCard` with a "Publish to GitHub" button appears on completion.

## Changes

- **GenerationProgress**: added `data-testid="a2ui-GenerationProgress"` and `data-step`
  attributes for E2E testability.
- **SummaryCard**: added `data-testid="a2ui-SummaryCard"` for E2E testability.
- **SSE write_file interception**: `tool_done` events for `core.write_file` now carry
  `path` and `content`, routed to the VFS and auto-opened in the editor pane.
- **Security**: write_file payloads are validated (path normalization, traversal rejection,
  5 MB cap) before writing to VFS per Zapp's conditional approval.
- **E2E test**: `phase-c-codesmith-progress.spec.ts` asserts the full flow — progress
  card, file editor routing, SummaryCard + publish button, zero CodeBlocks.

Closes azure-management-and-platforms/kickstart#7.
