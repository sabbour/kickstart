## Problem Statement

Users reported:
- Markdown formatting not appearing in chat
- "Empty stream response" error when chatting
- /login route not properly integrated with MSAL auth flow
- Missing user avatar in chat interface

## Root Causes

1. **Markdown Bug:** renderText() using textContent instead of renderMarkdown()
2. **Streaming Bug:** mergeChunk not handling server's 'content' field correctly
3. **Error Handling:** Error messages not surfacing actual server error details
4. **Login Flow:** /login route configured for SWA server redirect but app uses client-side MSAL
5. **Avatar:** No user profile image in chat UI

## Solutions Implemented

### Chat Fixes (Coordinator)
- A2UI renderText now calls renderMarkdown() for proper formatting
- mergeChunk updated to parse server's content field
- Error handler now displays actual server error strings
