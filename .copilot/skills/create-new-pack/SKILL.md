# Create a New Pack

**Confidence:** high  
**Last updated:** 2026-04-20  
**Use when:** Adding a new domain-specific pack to extend Kickstart (e.g., a new cloud provider, service integration, or vertical)

## What is a Pack?

A **Pack** is a self-contained, domain-specific module that extends Kickstart with agents, skills, tools, user actions, components, and manifests. Each pack is a standalone Node.js package under `packages/pack-*`.

**Examples in the repo:**
- **pack-core** — Domain-neutral agents, skills, tools (used by all packs)
- **pack-azure** — Azure-specific agents, skills, tools, components
- **pack-github** — GitHub-specific tools and user actions
- **pack-aks-automatic** — AKS automation agents and skills

## Pack Directory Structure

```
packages/pack-{name}/
├── src/
│   ├── index.ts               # Main export
│   ├── {name}-pack.ts         # Pack metadata + registration
│   ├── server-manifest.ts     # Build output listing agents, skills, tools
│   ├── agents/                # Agent definitions (*.ts)
│   │   └── {agent-name}.ts
│   ├── skills/                # Skill definitions (skill.ts files per skill)
│   │   └── {skill-name}/
│   │       └── skill.ts
│   ├── tools/                 # Tool implementations
│   │   └── {tool-name}.ts
│   ├── components/            # React components (A2UI or smart)
│   │   └── {ComponentName}.tsx
│   ├── user-actions/          # User actions (async work triggered from UI)
│   │   └── {action-name}.ts
│   ├── services/              # API clients, integrations
│   │   └── {service}.ts
│   └── guardrails/            # Content filters, safety checks
│       └── {guardrail}.ts
├── __tests__/                 # Unit tests (vitest)
│   └── {file}.test.ts
├── package.json               # Deps, scripts, exports
├── tsconfig.json              # TypeScript config
├── CHANGELOG.md               # Changelog (use changesets)
└── README.md                  # Pack documentation
```

## Steps

### Step 1: Create the Package

Create directory: `packages/pack-{name}/`

File: `packages/pack-{name}/package.json`

```json
{
  "name": "@kickstart/pack-{name}",
  "version": "1.0.0",
  "description": "{Descriptive name} — agents, skills, tools, and components for {domain}",
  "type": "module",
  "main": "dist/index.js",
  "types": "dist/index.d.ts",
  "exports": {
    ".": {
      "import": "./dist/index.js",
      "types": "./dist/index.d.ts"
    },
    "./server-manifest": {
      "import": "./dist/server-manifest.js",
      "types": "./dist/server-manifest.d.ts"
    }
  },
  "scripts": {
    "build": "tsc",
    "clean": "rm -rf dist tsconfig.tsbuildinfo",
    "test": "vitest run"
  },
  "dependencies": {
    "@kickstart/harness": "*",
    "@openai/agents": "*",
    "zod": "^3.0.0"
  },
  "devDependencies": {
    "@testing-library/react": "^14.0.0",
    "typescript": "^5.0.0",
    "vitest": "^1.0.0"
  }
}
```

### Step 2: Create Pack Metadata

File: `packages/pack-{name}/src/{name}-pack.ts`

```ts
import { PackDefinition } from '@kickstart/harness';

export const {name}Pack: PackDefinition = {
  name: '@kickstart/pack-{name}',
  version: '1.0.0',
  description: '{Domain-specific agents, skills, tools}',
  agents: [
    // List agents exported by this pack
    // { name: '{agent-name}', path: './agents/{agent-name}.js' },
  ],
  skills: [
    // List skills exported by this pack
    // { name: '{skill-name}', path: './skills/{skill-name}/skill.js' },
  ],
  tools: [
    // List tools exported by this pack
    // { name: '{tool-name}', path: './tools/{tool-name}.js' },
  ],
};
```

### Step 3: Create Server Manifest

File: `packages/pack-{name}/src/server-manifest.ts`

```ts
// Generated by build process — lists all agents, skills, tools, components
// This manifest tells the harness what's available in this pack

import type { ServerManifest } from '@kickstart/harness';

export const manifest: ServerManifest = {
  agents: [
    // { name: '{name}', path: '/agents/{name}.js' },
  ],
  skills: [
    // { name: '{name}', path: '/skills/{name}/skill.js' },
  ],
  tools: [
    // { name: '{name}', path: '/tools/{name}.js' },
  ],
  components: [
    // { name: '{ComponentName}', path: '/components/{ComponentName}.js' },
  ],
};
```

### Step 4: Create Main Export

File: `packages/pack-{name}/src/index.ts`

```ts
// Re-export pack metadata
export { {name}Pack } from './{name}-pack';

// Re-export all agents, skills, tools, components from subdirectories
// export { AgentName } from './agents/agent-name';
// export { skillName } from './skills/skill-name/skill';
// export { toolName } from './tools/tool-name';
// export { ComponentName } from './components/ComponentName';
```

### Step 5: Create TypeScript Config

File: `packages/pack-{name}/tsconfig.json`

```json
{
  "extends": "../../tsconfig.json",
  "compilerOptions": {
    "outDir": "./dist",
    "rootDir": "./src"
  },
  "include": ["src/**/*"],
  "exclude": ["**/*.test.ts", "__tests__/**/*"]
}
```

### Step 6: Create README and Changelog

File: `packages/pack-{name}/README.md`

```markdown
# @kickstart/pack-{name}

{Domain-specific agents, skills, tools, and components for {domain}}.

## What's in this pack

- **Agents:** {List key agents}
- **Skills:** {List key skills}
- **Tools:** {List key tools}
- **Components:** {List key A2UI/smart components}

## Installation

```bash
npm install @kickstart/pack-{name}
```

## Quick Start

{Short example of using pack}

## Documentation

See [docs](https://sabbour.me/kickstart/docs) for full docs.
```

File: `packages/pack-{name}/.changeset/init.md` (create one after first commit)

```markdown
---
"@kickstart/pack-{name}": minor
---

Initial release: {brief description of pack}
```

### Step 7: Register in Root package.json

Edit: `/package.json`

Add to `"workspaces"` array:

```json
"workspaces": [
  "packages/pack-{name}",
  "packages/*"
]
```

### Step 8: Update Root tsconfig

Edit: `/tsconfig.json`

Add to `"compilerOptions.paths"`:

```json
"@kickstart/pack-{name}": ["packages/pack-{name}/src"],
"@kickstart/pack-{name}/*": ["packages/pack-{name}/src/*"]
```

### Step 9: Add to Harness Registry (Optional)

If the pack should be auto-discovered by the harness, register it in:

Edit: `packages/harness/src/runtime/loader-pack.ts`

```ts
import { {name}Pack } from '@kickstart/pack-{name}';

const packs = [
  // ... existing packs
  {name}Pack,
];
```

## Build and Test

### Before you commit:

1. **Create pack structure:** Run `npm install` from repo root (workspaces auto-link)
2. **Build the pack:** `npm run build -w @kickstart/pack-{name}`
3. **Run tests:** `npm test -w @kickstart/pack-{name}`
4. **Full validation:** `npm run build && npm test`
5. **Check exports:** Verify `dist/index.js` and `dist/index.d.ts` exist and export correctly

### Checklist:

- [ ] Package created under `packages/pack-{name}/`
- [ ] `package.json` configured with correct name and exports
- [ ] `src/{name}-pack.ts` metadata file created
- [ ] `src/server-manifest.ts` created (can be minimal)
- [ ] `src/index.ts` re-exports from subdirectories
- [ ] `tsconfig.json` extends root config
- [ ] `README.md` documents the pack
- [ ] Registered in root `package.json` workspaces
- [ ] Registered in root `tsconfig.json` paths
- [ ] Builds without errors: `npm run build -w @kickstart/pack-{name}`
- [ ] Tests pass: `npm test -w @kickstart/pack-{name}`

## Adding Agents to Your Pack

Once the pack structure is set up:

1. Create `src/agents/{agent-name}.ts` with your agent definition
2. Add to the `agents` array in `src/{name}-pack.ts`
3. Re-export from `src/index.ts`
4. Update `src/server-manifest.ts` with the agent entry

Agents within packs follow the same patterns as pack-core agents — they have a charter, implement skills, use tools, and dispatch actions.

## Adding Skills to Your Pack

Same pattern:

1. Create `src/skills/{skill-name}/skill.ts`
2. Add to `skills` array in pack metadata
3. Re-export from `src/index.ts`
4. Update server manifest
